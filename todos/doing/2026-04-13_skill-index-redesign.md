# Skill Index Redesign (v2 — post codex review)

Redesign the agent-skill-index mechanism so platform-specific subagents can
discover relevant skills reliably across machines, with graceful degradation
when optional dependencies (Axiom plugin, user-built skills) are absent.

**Scope: iOS only for v1.** Other platforms migrate later if/when they need
the same treatment. See "Why iOS-only" below.

## Problem with current design

Current `skills/agent-skill-index/references/{platform}.md` mixes three
unrelated content sources into a single static file:

1. **User-level skills** (e.g. legacy `ios-patterns-hana`) — only exist on one
   maintainer machine
2. **Axiom plugin skills** (200+ entries with `axiom-` prefix) — only available
   when the separate `axiom-marketplace` plugin is installed
3. **Hand-curated guidance** ("Common Combinations") — maintainer-curated picks

Consequences:

- Anyone forking solopreneur without the maintainer's exact setup hits dead links for
  most entries
- No mechanism to detect what's actually available on the machine
- Adding a new self-built skill requires manually editing the index file
- Axiom version upgrades that add/remove skills require manual sync
- The index file lives under `skills/` but is not a skill itself
- Subagent must Glob to find the index even though most useful content
  (curated picks) is what should *always* be in the subagent's context

## Why iOS-only for v1

Codex review confirmed (`web-index:12`, `python-index`, `llm-index`) that
non-iOS platforms have minimal current value:

- `web/python/llm` indices contain ~1 skill each, mostly duplicated
- `python-dev` agent doesn't even consume the skill-index flow currently
- `nextjs-dev` is missing from the original plan but documented in README

Real pain point is iOS — 5 user skills + 200+ Axiom entries + dead-link risk.
Build the system for iOS, prove it works, then port to other platforms only
if the use case justifies it.

## Design

### Two content tiers

| Tier | Content | Lives in | Source of truth |
|------|---------|----------|-----------------|
| Curated | Hand-picked 10-20 iOS skills with one-line descriptions | `agents/ios-dev.md` (inline section) | Plugin git, version-controlled |
| Extended | All `~/.claude/skills/*/SKILL.md` matching `platform: ios` frontmatter, plus full Axiom catalog if installed, plus missing-plugin warnings | `~/.claude/solopreneur/skill-index/ios.md` | Generated by build script |

### Single source of skill knowledge: agent md

Critical insight from review: `specialist-review`, `preflight`, `todos-review`
all dispatch the same `ios-dev` subagent type. The subagent's system prompt
is `agents/ios-dev.md`. So if curated + discovery instructions live in agent
md, **all consumers automatically inherit the new behavior** without per-skill
modification.

This means:

- `specialist-review` no longer needs its own skill-index lookup logic
  (currently `specialist-review:82-92`) — delete it
- Hardcoded `~/.claude/skills/{skill-name}/SKILL.md` path
  (`specialist-review:117`) — delete it; agent md owns path resolution
- Other future consumers also benefit transparently

### Agent md changes (`agents/ios-dev.md`)

Add two sections:

```markdown
## Curated Skills

For any iOS task, always consider these. Skill names are clickable in the
extended index — read SKILL.md for any that matches your task.

- legacy `ios-patterns-hana` (user) — SwiftUI + i18n conventions
- `axiom-swiftui-performance` (axiom) — SwiftUI performance optimization
- ... (10-20 hand-picked entries)

Path resolution:
- `(user)` skills are at `~/.claude/skills/<name>/SKILL.md`
- `(axiom)` skills are at `~/.claude/plugins/cache/axiom-marketplace/axiom/<version>/skills/<name>/SKILL.md`
  (use `ls ~/.claude/plugins/cache/axiom-marketplace/axiom/` to find the version)

## Extended Discovery

Before producing any output (code, review, recommendation), Read:
`~/.claude/solopreneur/skill-index/ios.md`

This file contains auto-discovered user skills and the full Axiom catalog
that aren't in the curated list above. It includes the resolved path for
each skill so you can Read directly.

If the file does not exist, ask the user to run `/rebuild-skill-index`,
then fall back to context7 + built-in knowledge.
```

Note the trigger phrase change (Codex I1): "Before producing any output"
covers code-writing AND review-only consumers (`preflight`, `todos-review`,
`specialist-review`).

Curated bullet format spec for the parser:

```
- `<skill-name>` (<source>) — <description>
```

Where `<source>` is `user` or `axiom`. Parser regex:
`^- \`([^\`]+)\` \((user|axiom)\) — `

### Build script (`scripts/rebuild-skill-index.sh`)

Inputs:
- `agents/ios-dev.md` — parses `## Curated Skills` section to extract skill
  names (used as dedup blacklist)
- `~/.claude/skills/*/SKILL.md` — reads frontmatter for `platform: ios` tag
- `~/.claude/plugins/cache/axiom-marketplace/axiom/<version>/skills/` — note
  the extra `axiom/` layer (Codex C3); enumerates if exists; if absent,
  emits a warning

Output: `~/.claude/solopreneur/skill-index/ios.md`

Output structure (every entry includes resolved path — Codex C2):

```markdown
# iOS Skills Index — Extended
Generated: 2026-04-13T12:34:56+08:00

## Auto-discovered user skills (platform: ios)
- `ios-kavsoft-reference` — 140+ SwiftUI examples
  Path: ~/.claude/skills/ios-kavsoft-reference/SKILL.md
- ... (any other matching skills not in curated)

## Axiom plugin skills (axiom-marketplace v2.37.0)
- `axiom-ios-ui` — SwiftUI, UIKit, layout, navigation
  Path: ~/.claude/plugins/cache/axiom-marketplace/axiom/2.37.0/skills/axiom-ios-ui/SKILL.md
- `axiom-swiftui-architecture` — MVVM patterns
  Path: ~/.claude/plugins/cache/axiom-marketplace/axiom/2.37.0/skills/axiom-swiftui-architecture/SKILL.md
- ... (alphabetical, no subcategorization — Codex I5)

## Missing
(empty if all good; otherwise warnings like "Axiom not installed — install at https://...")
```

Skills are listed alphabetically. No `### UI` / `### Data` subheadings —
Axiom skill frontmatter doesn't carry category info, so any subcategorization
would be hand-coded and would drift over time.

Script language: bash (matches existing scripts in solopreneur). Uses `jq`
for JSON, `awk`/`grep` for markdown parsing.

Script must `mkdir -p ~/.claude/solopreneur/skill-index/` before writing.

### Slash command (`commands/rebuild-skill-index.md`)

Wraps script invocation. Prints summary of what was generated. Surfaces
warnings inline.

### Self-built skill discovery — migration required

Codex I3: existing iOS skills don't have `platform: ios` frontmatter:

- `~/.claude/skills/ios-patterns-hana/SKILL.md`
- `~/.claude/skills/ios-iphone-apps/SKILL.md`
- `~/.claude/skills/ios-kavsoft-reference/SKILL.md`
- `~/.claude/skills/ios-swiftui-nav-ref/SKILL.md`
- `~/.claude/skills/ios-swiftui-26-ref/SKILL.md`

Without the tag, auto-discovery returns empty. Migration step needed
**before** running rebuild-skill-index for the first time:

```bash
# Add `platform: ios` to each of the 5 SKILL.md files above
```

This is a one-time manual edit (not script-automated — risk of breaking
custom frontmatter).

## Explicit non-goals (v1)

- No Android/Web/Python/LLM/NextJS support — iOS only
- No automatic rebuild on subagent invocation (manual `/rebuild-skill-index`)
- No pre-commit hook (output is per-machine, not committed)
- No CLI wrapper like `gstack-config`
- No XDG `~/.solopreneur/` home directory (use `~/.claude/solopreneur/`)
- No marker comments in agent md (`<!-- CURATED:START/END -->`) — heading-based
  parsing is sufficient
- No keyword-based auto-discovery for skills lacking `platform:` frontmatter
- No Axiom skill subcategorization (alphabetical only)
- No `solopreneur.json` config — fixed defaults work for iOS-only scope
  (revisit when adding more platforms)

## Risks

- **Heading rename breaks script:** if `## Curated Skills` is renamed in
  agent md, parser silently produces wrong blacklist. Mitigation: script logs
  warning if it parses zero curated entries from a non-empty file
- **Maintainer edits curated frequently and forgets to rerun script:** extended
  file becomes stale (still lists newly-curated skill as auto-discovered).
  Mitigation: subagent sees both lists; cost is one extra duplicate entry
- **Axiom path layer (`axiom-marketplace/axiom/<version>/`) changes upstream:**
  hardcoded path breaks. Mitigation: script does `ls` to find version dir,
  doesn't assume single fixed depth
- **Specialist-review subagent assumes agent md is loaded:** verified —
  Claude Code subagents always load their agent type's md as system prompt
- **Cross-machine portability:** generated extended file is per-machine by
  design. Plugin ships curated only.
